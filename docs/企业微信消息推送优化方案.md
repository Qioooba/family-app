# 企业微信消息推送优化方案

## 📅 讨论时间
2026-03-12

## 🎯 目标
在微信企业微信插件中，实现点击消息先展示详情，再选择是否跳转小程序。

---

## ✅ 方案确定

### 消息类型
**全部使用图文消息（news）**

### 流程
```
微信消息列表
    ↓ 点击
H5详情页（显示完整任务信息）
    ↓ 用户点击按钮
打开小程序 / 关闭
```

### 消息格式
```json
{
  "msgtype": "news",
  "news": {
    "articles": [{
      "title": "🏠 新任务：记得买牛奶",
      "description": "来自：陶陶\n时间：今天 18:00\n\n点击查看详情",
      "url": "https://qioba.cn:8443/notify-detail.html?type=task&id=123&token=xxx",
      "picurl": "https://qioba.cn:8443/logo.png"
    }]
  }
}
```

## ⚠️ 关键限制确认（Kimi补充）

### 微信App企业微信插件的支持情况

| 消息类型 | 微信插件支持 | 说明 |
|---------|------------|------|
| **图文消息（news）** | ✅ **支持** | **唯一可行方案** |
| 模板卡片（template_card） | ❌ 不支持 | 微信插件（微工作台）不支持展示 |
| 文本消息（text） | ✅ 支持 | 过于简单，无法展示完整详情 |
| 文本卡片（textcard） | ✅ 支持 | 可跳转URL，但无图片展示 |
| 小程序通知（miniprogram_notice） | ✅ 支持 | 点击直接进小程序，无详情页 |

### 为什么只能用图文消息？

**官方限制**：微信App里的"企业微信插件"本质上就是**微工作台**，官方文档明确指出：
> 微工作台不支持展示模板卡片消息

因此：
- ❌ 模板卡片会降级显示或无法显示
- ✅ 必须使用基础消息类型：文本、图文、图片、文件等

### 最终结论

**图文消息（news）是唯一可行且最优的方案**：

```
微信App企业微信插件
    ↓ 收到图文消息
显示：应用头像 + 标题 + 简介
    ↓ 点击消息
打开 url 对应的 H5 详情页
    ↓ 查看详情
点击按钮调用 launchMiniprogram 打开小程序
```
