## 一、关键修改

### 1.1 推送优先级修改 ✅

**修改前**: 优先内部成员推送
```java
// 优先使用内部成员推送（小程序卡片）
if (identity.hasWorkUserId()) {
    sendReminderMiniappCardToInternalUser(...); // mpnews内置文章
    return;
}
// 只有外部联系人ID
if (identity.hasExternalUserId()) {
    sendToExternalUser(...); // news图文
}
```

**修改后**: 优先外部联系人推送（微信个人号）
```java
// 优先使用外部联系人推送（推送到微信个人号）
if (identity.hasExternalUserId()) {
    try {
        sendToExternalUser(token, identity.externalUserId, message);
        return;
    } catch (Exception e) {
        log.warn("外部联系人推送失败，尝试内部成员, userId={}", message.getTargetUserId(), e);
        // 外部失败时回退到内部
    }
}

// 使用内部成员推送（企业微信内部）
if (identity.hasWorkUserId()) {
    sendReminderMiniappCardToInternalUser(token, identity.workUserId, message);
}
```

- `40003`: 外部联系人ID无效

- **必须48小时内有过互动**才能发送消息

**错误码**: `48002`
**错误信息**: "外部联系人48小时内无互动，无法发送消息"

### 3.2 解决方案

**方案A**: 让陶陶提前互动（推荐）


- 如果外部推送失败，自动回退到内部推送
- 内部推送无48小时限制

## 七、48小时互动解决方案详解

### 7.1 问题说明

企业微信为保护用户，限制外部联系人消息：
- 只有48小时内互动过的客户才能接收消息
- 这是企业微信的强制规则，无法绕过

### 7.2 互动方式（任一即可）

**方式1**: 陶陶主动给小助手发消息
```
微信中打开企业微信小助手聊天
↓
发送任意文字（如"1"或"早安"）
↓
互动时间更新，48小时倒计时重新开始
```

**方式2**: 点击小程序
```
打开家庭小程序
↓
点击"我的"或任意页面
↓
自动更新互动时间
```

**方式3**: 点击历史推送
```
点击之前收到的小助手消息
↓
进入H5详情页
↓
互动时间更新
```

### 7.3 最佳实践

建议在**早晨8点前1小时内**完成互动，确保推送时48小时限制有效。

例如：
- 7:30 陶陶给小助手发消息
- 8:00 系统自动推送提醒 ✅
