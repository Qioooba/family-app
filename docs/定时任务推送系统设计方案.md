## 🗄️ 数据库表设计

### 表1：定时任务配置表（schedule_job）
```sql
CREATE TABLE schedule_job (
    id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '主键ID',
    job_code VARCHAR(50) NOT NULL COMMENT '任务编码：MORNING_PUSH / DEADLINE_REMIND',
    job_name VARCHAR(100) NOT NULL COMMENT '任务名称：如"早安推送"',
    job_group VARCHAR(50) DEFAULT 'DEFAULT' COMMENT '任务分组',
    
    -- 时间配置
    cron_expression VARCHAR(100) COMMENT 'Cron表达式（优先）',
    execute_time VARCHAR(20) COMMENT '执行时间，如 08:00（Cron为空时使用）',
    
    -- 工作日/节假日配置
    work_days_only TINYINT DEFAULT 0 COMMENT '仅工作日推送：0-否 1-是',
    exclude_holiday TINYINT DEFAULT 0 COMMENT '排除节假日：0-否 1-是',
    
    -- 推送范围配置（JSON格式，灵活扩展）
    push_config JSON COMMENT '推送配置：{
        "pushType": "ALL",           // ALL-全部 MEMBER-指定成员
        "targetUserIds": [7, 16],    // 指定用户ID列表
        "pushContent": {             // 内容配置
            "includeWeather": true,  // 包含天气
            "includeTodo": true,     // 包含待办
            "greetingType": "auto"   // auto-自动 custom-自定义
        }
    }',
    
    -- 任务特定配置
    job_config JSON COMMENT '任务特定配置：{
        // 早安推送配置
        "advanceMinutes": 0,
        // 到期提醒配置
        "remindBeforeMinutes": 30,   // 提前多少分钟提醒
        "remindTypes": ["ASSIGNED", "CREATED"]  // 提醒哪些类型的待办
    }',
    
    status TINYINT DEFAULT 1 COMMENT '状态：0-停用 1-启用 2-执行中',
    description VARCHAR(500) COMMENT '任务描述',
    
    -- 执行记录
    last_execute_time DATETIME COMMENT '上次执行时间',
    last_execute_result VARCHAR(20) COMMENT '上次执行结果：SUCCESS / FAILED',
    last_execute_msg VARCHAR(500) COMMENT '上次执行结果消息',
    next_execute_time DATETIME COMMENT '下次执行时间',
    
    -- 管理员信息
    create_by BIGINT COMMENT '创建人ID',
    update_by BIGINT COMMENT '更新人ID',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_job_code (job_code),
    INDEX idx_status (status),
    INDEX idx_next_time (next_execute_time)
) COMMENT='定时任务配置表';
```

### 表2：定时任务执行日志（schedule_log）
```sql
CREATE TABLE schedule_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    job_id BIGINT NOT NULL COMMENT '任务ID',
    job_code VARCHAR(50) COMMENT '任务编码',
    job_name VARCHAR(100) COMMENT '任务名称',
    
    execute_time DATETIME COMMENT '计划执行时间',
    actual_execute_time DATETIME COMMENT '实际执行时间',
    finish_time DATETIME COMMENT '完成时间',
    
    status VARCHAR(20) COMMENT '状态：SUCCESS / FAILED / PARTIAL',
    push_total INT DEFAULT 0 COMMENT '应推送人数',
    push_success INT DEFAULT 0 COMMENT '推送成功人数',
    push_fail INT DEFAULT 0 COMMENT '推送失败人数',
    
    execute_result TEXT COMMENT '执行结果详情（JSON）',
    error_msg TEXT COMMENT '错误信息',
    
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_job_id (job_id),
    INDEX idx_time (create_time),
    INDEX idx_status (status)
) COMMENT='定时任务执行日志表';
```

### 表3：用户定时任务设置（schedule_user_setting）
```sql
CREATE TABLE schedule_user_setting (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    job_code VARCHAR(50) NOT NULL COMMENT '任务编码',
    
    enabled TINYINT DEFAULT 1 COMMENT '是否启用：0-否 1-是',
    
    -- 个性化配置
    custom_config JSON COMMENT '个性化配置：{
        "advanceMinutes": 30,        // 提前提醒分钟数（到期提醒）
        "remindBeforeDeadline": true, // 到期前提醒
        "remindOnAssign": true,       // 被指派时提醒
        "quietHoursStart": "22:00",   // 免打扰开始时间
        "quietHoursEnd": "08:00"      // 免打扰结束时间
    }',
    
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_user_job (user_id, job_code),
    INDEX idx_user (user_id)
) COMMENT='用户定时任务个性化设置表';
```

## 📋 任务类型详细设计

### 任务1：早安推送（MorningPushJob）

**任务编码**：`MORNING_PUSH`

**默认配置**：
- 执行时间：08:00
- 仅工作日：是
- 推送人员：全部家庭成员
- 内容组成：早安问候 + 天气 + 今日待办列表

**执行流程**：
```
1. 查询schedule_job，获取配置
2. 判断是否工作日（查holiday_config表）
   - 节假日 → 跳过
   - 工作日 → 继续
3. 查询推送目标用户（根据push_config.targetUserIds）
4. 遍历用户：
   a. 查询用户今日待办（task表，今天截止且未完成）
   b. 查询用户天气（根据用户城市）
   c. 组装图文消息
   d. 企业微信推送
5. 记录执行日志
```

**消息示例**：
```
标题：🌅 早安齐军！今日3项待办
封面：天气图标
描述：
☀️ 徐州 18°C 晴

📋 今日待办：
• 买牛奶（18:00）
• 接孩子（16:30）
• 取快递（14:00）

💪 新的一天，加油！
```

### 任务2：到期提醒（DeadlineRemindJob）

**任务编码**：`DEADLINE_REMIND`

**默认配置**：
- 执行频率：每5分钟（Cron: `0 */5 * * * ?`）
- 提前提醒：30分钟
- 推送人员：任务执行人（assignee）

**执行流程**：
```
1. 查询所有未完成的待办（status != 2）
2. 筛选出快到期的：
   当前时间 + advanceMinutes >= 截止时间
   且 当前时间 < 截止时间（未过期）
3. 检查是否已提醒（避免重复）：
   - 查schedule_log，今天是否已提醒过该任务
   - 或查Redis缓存
4. 遍历需要提醒的任务：
   a. 查询任务执行人
   b. 检查用户是否开启提醒（schedule_user_setting）
   c. 组装提醒消息
   d. 企业微信推送
5. 记录执行日志
```

**消息示例**：
```
标题：⏰ 待办提醒：买牛奶
描述：
任务：买牛奶
截止时间：今天 18:00
剩余时间：30分钟

点击查看详情
```
