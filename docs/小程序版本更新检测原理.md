用户问：
1. **如何检测用户版本低于正式版？**

## 一、版本检测原理

### 1.1 谁来做版本对比？

**不是我们的代码对比，是微信客户端对比！**

```
用户手机上的小程序A（旧版本：1.0.2）
          ↓
    启动时询问微信服务器
          ↓
微信服务器（最新版本：1.0.3）
          ↓
    返回：有新版本！
          ↓
用户手机自动下载新版本（后台静默下载）
          ↓
下载完成 → 触发 onUpdateReady → 提示用户重启
```

### 1.2 代码中的三个回调

```javascript
const updateManager = wx.getUpdateManager()

// 1. 检测是否有新版本（启动时自动调用）
updateManager.onCheckForUpdate(function (res) {
    console.log(res.hasUpdate)  // true-有新版本 false-无新版本
})

// 2. 新版本下载完成时触发
updateManager.onUpdateReady(function () {
    // 弹出提示框，让用户确认重启
    wx.showModal({
        title: '更新提示',
        content: '新版本已准备好，是否重启应用？',
        success: function (res) {
            if (res.confirm) {
                // 强制重启，使用新版本
                updateManager.applyUpdate()
            }
        }
    })
})

// 3. 新版本下载失败时触发
updateManager.onUpdateFailed(function () {
    console.log('下载失败')
})
```

### 1.3 版本对比的时机

| 时机 | 行为 |
|------|------|
| **冷启动**（完全关闭后打开） | 必定检测新版本 |
| **热启动**（后台切换到前台） | 可能会检测 |
| **微信定期检测** | 每隔几小时自动检测 |

## 二、缓存检测与清理

### 2.1 什么是小程序缓存？

小程序缓存包括：
- ✅ 代码包（WXML/WXSS/JS/JSON）
- ✅ 本地存储（wx.setStorageSync）
- ✅ 图片/文件缓存
- ✅ 授权信息

### 2.2 版本更新是否自动清理缓存？

**代码包缓存**：✅ 自动清理（新版本下载后旧代码失效）  
**本地存储（Storage）**：❌ 不会自动清理！

### 2.3 我们的代码如何清理Storage？

```javascript
const clearExpiredCache = () => {
    // 1. 读取上次清理时间
    const timestamp = uni.getStorageSync('cache_clear_time')
    const now = Date.now()
    
    // 2. 判断是否需要清理（超过1小时或强制清理标志）
    if (!timestamp || (now - timestamp > 3600000) || uni.getStorageSync('force_clear')) {
        
        // 3. 保留关键配置，清理其他所有数据
        const keepKeys = ['family_invite_code', 'cache_clear_time']
        const allKeys = uni.getStorageInfoSync().keys || []
        
        allKeys.forEach(key => {
            if (!keepKeys.includes(key)) {
                uni.removeStorageSync(key)  // 删除缓存
            }
        })
        
        // 4. 记录本次清理时间
        uni.setStorageSync('cache_clear_time', now)
    }
}
```

### 2.4 清理策略

| 数据类型 | 是否清理 | 说明 |
|---------|---------|------|
| token | ✅ 清理 | 强制重新登录 |
| userInfo | ✅ 清理 | 重新获取 |
| 页面数据 | ✅ 清理 | 重新加载 |
| invite_code | ❌ 保留 | 邀请码不能丢 |
| cache_clear_time | ❌ 保留 | 记录清理时间 |

## 四、关键代码解析

### 4.1 `wx.getUpdateManager()` 是什么？

这是**微信提供的原生API**，作用是：
- 管理小程序版本更新
- 监听更新事件
- 控制更新行为

**文档**：https://developers.weixin.qq.com/miniprogram/dev/api/base/update/UpdateManager.html

### 4.2 为什么能强制更新？

```javascript
updateManager.applyUpdate()
```

这行代码的作用：
- ✅ 立即停止当前小程序
- ✅ 销毁所有页面
- ✅ 重新启动，加载最新代码包
- ✅ 相当于：关闭小程序 → 重新打开

### 4.3 用户能不能拒绝更新？

**技术上可以，但实际上会强制更新**：

```javascript
wx.showModal({
    content: '新版本已准备好，是否重启应用？',
    // 没有 cancelText 和 cancelColor 参数
    // 用户点击"取消"只是关闭弹窗
})
```

**注意**：如果用户点击"取消"，小程序不会重启，但：
- 新版本已经下载好了
- 下次启动时会自动使用新版本
- 用户无法一直使用旧版本

## 五、常见问题

### Q1：为什么有时候检测不到新版本？

**可能原因**：
1. 微信客户端有缓存（30分钟内不会重复检测）
2. 用户网络不好
3. 微信服务器延迟

**解决方案**：
- 用户下拉小程序（触发刷新）
- 用户完全关闭微信再打开
- 等待10-30分钟后重试

### Q2：更新后数据丢失了怎么办？

**更新只影响代码，不影响Storage数据**

但我们的代码会清理Storage（超过1小时），如果不想让数据丢失：

```javascript
// 把需要保留的key加入白名单
const keepKeys = [
    'family_invite_code', 
    'cache_clear_time',
    'user_settings',      // 新增：用户设置
    'game_records'        // 新增：游戏记录
]
```

### Q3：能不能静默更新，不提示用户？

**不可以！** 微信小程序设计如此：
- 必须用户确认后才能重启
- 避免用户操作被打断

但可以在 `onUpdateReady` 中不弹窗，这样：
- 用户继续使用旧版本
- 下次冷启动时自动使用新版本

## 六、总结

| 问题 | 答案 |
|------|------|
| 如何检测版本？ | 微信客户端自动对比服务器版本 |
| 如何强制更新？ | `updateManager.applyUpdate()` |
| 如何清理缓存？ | 手动清理 `uni.removeStorageSync` |
| 更新时机？ | 冷启动时 + 微信定期检测 |
| 用户能否拒绝？ | 可以取消弹窗，但下次启动仍是新版本 |

---

**一句话总结**：  
微信自动检测版本 → 后台下载 → 提示用户 → 用户确认 → 强制重启 → 使用新版本  
Storage缓存需要代码手动清理！
