## 一、后端代码审核

### 1.1 提醒模块 (ReminderScheduleService.java)

#### ✅ 优点
- 使用分布式锁防止重复执行
- 完善的异常处理机制
- 支持多种频率类型（ONCE/DAILY/WEEKLY/MONTHLY/YEARLY/INTERVAL）
- 工作日跳过逻辑正确

#### ⚠️ 发现的问题

| 问题 | 严重程度 | 位置 | 说明 |
|------|---------|------|------|
| 白名单硬编码 | 中 | 第106行 | `Set.of(7L, 16L)` 应该配置化 |
| 魔法数字 | 低 | 多处 | 如 100条限制、30天最大跳过等应配置化 |
| SQL 注入风险 | 低 | Mapper | 使用 `@Select` 注解拼接 SQL |
| 日志级别 | 低 | 多处 | 调试日志应使用 DEBUG 级别 |

#### 🔧 建议优化
```java
// 1. 白名单配置化
@Value("${reminder.push.whitelist:7,16}")
private Set<Long> whitelist;

// 2. 常量提取
private static final int MAX_REMINDERS_PER_SCAN = 100;
private static final int MAX_SKIP_DAYS = 30;
```

### 1.2 企业微信推送 (WechatWorkService.java)

#### ✅ 优点
- 支持多种消息类型（text/news/mpnews/miniprogram）
- 完善的错误处理和重试机制
- 媒体文件上传功能完整

#### ⚠️ 发现的问题

| 问题 | 严重程度 | 位置 | 说明 |
|------|---------|------|------|
| 敏感信息硬编码 | 高 | config | corpId/secret 应加密存储 |
| 重试机制不完善 | 中 | sendReminderWithQrCodeAsync | 固定延迟，应指数退避 |
| 空指针风险 | 中 | 多处 | `getMiniProgramAppId()` 可能返回 null |
| 资源未关闭 | 低 | uploadMiniappQrImage | ByteArrayResource 应 try-with-resources |

#### 🔧 建议优化
```java
// 1. 敏感信息加密
@Value("${wechat.work.corpid:}")
private String corpId;

// 2. 指数退避重试
int delay = 1000 * (int)Math.pow(2, i);

// 3. 空值检查
String appId = getMiniProgramAppId();
if (appId == null || appId.isEmpty()) {
    throw new IllegalStateException("小程序AppID未配置");
}
```

### 1.3 数据库实体 (Reminder.java)

#### ✅ 优点
- 使用 MyBatis-Plus 注解
- 字段注释完整
- 支持自动填充

#### ⚠️ 发现的问题

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| 字段类型不一致 | 中 | `targetUserIds` 是 String(JSON)，应使用类型处理器 |
| 缺少索引注解 | 低 | 频繁查询的字段应加索引 |

  uni.$off('openAddReminder')

weekDays: Array.isArray(config.weekDays) ? config.weekDays : []

## 三、安全审核

### 3.1 高风险问题

| 问题 | 影响 | 建议 |
|------|------|------|
| 企业微信密钥硬编码 | 泄露风险 | 使用配置中心或加密存储 |
| SQL 拼接 | 注入风险 | 使用 MyBatis 参数绑定 |
| 无接口限流 | DDoS 风险 | 添加 RateLimiter |

### 3.2 中风险问题

| 问题 | 影响 | 建议 |
|------|------|------|
| 敏感操作无审计日志 | 无法追溯 | 添加操作日志表 |
| 会话超时时间过长 | 安全风险 | 缩短 token 有效期 |
| 前端源码暴露 | 逆向风险 | 使用代码混淆 |

## 四、性能审核

### 4.1 后端性能

| 问题 | 影响 | 建议 |
|------|------|------|
| 每分钟全表扫描 | 数据库压力 | 添加索引 + 分区表 |
| 媒体文件重复上传 | 存储浪费 | 添加文件哈希去重 |
| 无缓存 | 响应慢 | Redis 缓存热点数据 |

### 4.2 前端性能

| 问题 | 影响 | 建议 |
|------|------|------|
| 图片未压缩 | 包体积大 | 使用 CDN + WebP |
| 无懒加载 | 首屏慢 | 路由懒加载 |
| 列表无虚拟滚动 | 卡顿 | 长列表使用 virtual-scroll |
