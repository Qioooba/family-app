# Integrations

这里存放不属于前端主工程、也不属于后端主服务的独立集成单元。

## 目录说明

- `tencent-cloud-function/`
  - Node 版腾讯云函数
  - 用于独立部署的企业微信消息发送能力
- `tencent-scf/`
  - Python 版腾讯云函数
  - 与 Node 版用途接近，作为另一套运行时实现保留
- `wecom-vercel-api/`
  - 企业微信回调与可信域名验证的小型独立服务
  - 适用于部署到 Vercel 等平台
  - 当前作为主仓内普通目录维护，不再依赖嵌套 Git 元数据

## 为什么不放进 frontend 或 backend

- 它们有独立的运行环境和部署目标
- 不参与前端构建产物
- 不参与后端 JAR 打包
- 与主服务分开管理更清晰，也更不容易混淆部署边界
